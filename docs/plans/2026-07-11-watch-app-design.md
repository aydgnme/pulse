# Pulse — Apple Watch App Design (v1.1)

**Date:** 2026-07-11
**Status:** Approved; feasibility spike PASSED (`@bacons/apple-targets` v4.0.7 on
Expo SDK 57 generates a properly-embedded watch app target via a
`PBXFileSystemSynchronizedRootGroup`). Source written; **watchOS Xcode/EAS build
+ on-device QA still pending** — the watch target cannot be compiled in the
headless dev environment (SourceKit shows false-positive "not in scope" /
"unavailable in macOS" errors because it analyses files with the host SDK
outside the watch target's module).
**Scope:** Add a standalone watchOS companion to the existing Expo/React Native game.

## Context

Pulse is an Expo (SDK 57) React Native game. Core mechanic (`src/logic.ts`,
`src/Game.tsx`): a needle sweeps around a circle; the player taps when the
needle is inside a window around a target. Perfect/good grading, speed ramps
up, window shrinks, tension colour drifts mint → amber, best score persists,
ambient music with a mute toggle.

**Hard constraint:** watchOS cannot run React Native/Expo. The watch app must be
native **SwiftUI**. The RN game code is *not* reusable — the watch is a
from-scratch reimplementation. `logic.ts` (pure math) is ported to Swift.

This project uses Continuous Native Generation: `/ios` and `/android` are
gitignored and regenerated by `prebuild`/EAS. A watch target added by hand in
Xcode would be wiped on the next prebuild, so integration must survive CNG.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Integration | `@bacons/apple-targets` config plugin | Stays in Expo/CNG workflow; target survives prebuild. |
| App type | **Companion** watchOS app (paired with the Pulse iOS app) | `@bacons/apple-targets` does not support standalone watch apps — the target is distributed with the iOS app. No `WatchConnectivity` needed (no data sync). |
| Input | Auto-sweep needle; hit fired by **tap OR Digital Crown** | Same timing mechanic as phone; crown is an alternative trigger, both call one `registerHit()`. |
| Music | **Included**, foreground-only via `AVAudioPlayer` | Reuses `assets/audio/theme.wav`. **No background audio mode** on the watch target — avoids a watchOS repeat of the App Store 2.5.4 rejection. Mute toggle persisted. |
| Haptics | `WKInterfaceDevice` Taptic Engine | perfect → `.success`, good → `.click`, miss/death → `.failure`. |
| Persistence | `UserDefaults` (`pulse.best`, `pulse.muted`) | Local to watch. No phone↔watch sync (YAGNI). |
| Release | Separate `feature/watch-app` branch → version `1.1` | Independent of build 3 (the 2.5.4 fix), which ships first. |

## Architecture

```
targets/
  watch/
    expo-target.config.js      # @bacons/apple-targets target definition (type: watch)
    Info.plist                 # NO UIBackgroundModes
    PulseWatchApp.swift        # @main App / scene
    GameView.swift             # SwiftUI Canvas + TimelineView render + input
    GameModel.swift            # ObservableObject: phase, score, per-frame state
    PulseLogic.swift           # port of src/logic.ts (pure)
    AudioEngine.swift          # AVAudioPlayer loop, foreground-only, mute
    Haptics.swift              # WKInterfaceDevice wrapper
    Assets/theme.wav           # copied/referenced from assets/audio/theme.wav
app.json                       # add "@bacons/apple-targets" to plugins
```

### Rendering
- `TimelineView(.animation)` drives per-frame updates; `Canvas` draws the ring,
  the target arc slice, and the needle. Needle colour interpolates mint → amber
  from `tension(speed)`, matching the phone.
- Fixed-timestep angle integration in `GameModel` (mirrors the phone's rAF loop):
  `angle += dir * speed * dt`.

### Input → single path
- Full-screen tap gesture → `registerHit()`.
- `.digitalCrownRotation(...)`; when accumulated crown delta crosses a small
  threshold → `registerHit()` (then reset the accumulator). Any spin counts.

### Game loop / grading (ported from `logic.ts`)
`degreesUntilTarget`, `hitQuality`, `pointsFor`, `isMissed`, `nearMissDegrees`,
`speedAfterHit`, `windowAfterHit`, `tension`, `spawnTarget` — identical constants
(`TUNING`). On hit: score += points, speed/window step, spawn next target,
haptic + optional flash. On miss (needle leaves window unpressed) or wrong tap:
phase → `over`, death note, best updated.

### Audio
- `AudioEngine` loads `theme.wav`, loops at ~0.45 volume while `phase == playing`
  and not muted. Stops on background / menu / over. Uses `AVAudioSession`
  category `.playback` **without** background activation. No `UIBackgroundModes`.

## Screens
1. **Menu** — title, best score, "Tap or turn crown to start", mute toggle.
2. **Playing** — the dial, needle, target arc, live score.
3. **Over** — death note, score, best, "tap to retry" (with restart lockout like phone's `RESTART_LOCKOUT_MS`).

## Out of scope (YAGNI)
Share sheet, phone↔watch sync, complications, Always-On/ambient rendering,
independent tuning. Reassess after v1.1 ships.

## Risks & first step
`@bacons/apple-targets` is primarily exercised for widgets/extensions; a full
watchOS *app* target through it + EAS build may have rough edges. **Feasibility
spike first:** wire a minimal "hello world" watch target, run `prebuild`, and
confirm it builds on EAS before porting the game. **Fallback** if the plugin
can't carry a full watch app: eject to the bare workflow (commit `/ios`, add the
watch target directly in Xcode).

## Sequencing
1. Ship build 3 (2.5.4 fix) to App Store — independent, already building.
2. `feature/watch-app` branch: feasibility spike → port logic → SwiftUI game →
   audio/haptics → device QA → EAS build → submit as `1.1`.
